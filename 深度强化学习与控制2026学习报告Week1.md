
## 1 强化学习基础概念
### 1.1 两类人工智能任务
- **预测型任务**：
    - 根据数据预测所需输出——有监督学习
    - 生成数据实例——无监督学习
- **决策型任务**：
    - 在动态环境中采取行动——强化学习
     这种是序贯决策任务，指决策者序贯地做出一个个决策，并接续看到新的观测，直到最终任务结束
- **两者区别**：
    - 决策往往会使未来情况发生改变，因此决策者需要在未来的时间点做出进一步的决策
    - 预测仅仅产生一个针对输入数据的信号，并期望它和未来可观测到的信号一致，这不会使未来情况发生任何改变
### 1.2 决策任务的分类

| 环境特性                            | 白盒环境：变量和目标之间的关系 可以用具体公式表示       | 黑盒环境：变量和目标之间的关系无法用具体公式表示           |
| ------------------------------- | ------------------------------- | ---------------------------------- |
| 静态环境 <br>• 环境没有转移的状态<br>• 单步决策  | 运筹优化<br>• （混合整数）线性规划<br>• 非线形优化 | 黑盒优化<br>• 神经网络替代模型优化<br>• 贝叶斯优化    |
| 动态环境 <br>• 环境有可转移的状态 <br>• 多步决策 | 动态规划<br>• MDP直接求解<br>• 树、图搜索    | 强化学习 <br>• 策略优化 <br>• Bandits、序贯黑盒 |
|                                 |                                 |                                    |

### 1.3 强化学习
#### 1.3.1 定义
通过与环境交互学习以实现目标的计算方法。
#### 1.3.2 核心三要素
- 状态（State）/ 感知：感知环境状态
- 动作（Action）/ 行动：采取动作影响状态或者达到目标
- 奖励（Reward）/ 目标：随着时间推移最大化累积奖励
#### 1.3.3 交互流程
- 每一步$t$，智能体获得观测 $O_t$、奖励 $R_t$，执行动作 $A_t$
- 环境接收动作 $A_t$，返回 $O_{t+1}$、 $R_{t+1}$
#### 1.3.4 系统组成
##### 1.3.4.1 外部主体
1. **智能体（Agent）**：决策者、学习者，负责做动作、学习策略
2. **环境（Environment）**：外部世界，接收动作，返回观测、奖励
##### 1.3.4.2 内部要素
1. **历史（History）**： $H_t = \{O_1,R_1,A_1,\dots,O_t,R_t\}$
2. **状态（State）**： $S_t = f(H_t)$，是关于历史的函数，包含做未来决策所需信息
3. **策略（Policy）**：状态到动作的映射
   - 确定性策略： $a=\pi(s)$
   - 随机策略： $\pi(a|s)=P(A_t=a|S_t=s)$
4. **奖励（Reward）**： $R(s,a)$，立即衡量这一步动作的好坏
5. **价值函数（Value Function）**：对未来累积奖励的预测
6. **模型（Model）**：模拟环境的行为，即预测下一步的状态与奖励
#### 1.3.5 方法分类
##### 1.3.5.1 按“学习目标”
- 基于价值：知道什么是好的什么是坏的
    - SARSA（同策略 TD Learning 算法）
    - Q-Learning（异策略 TD Learning 算法）
    - DQN（深度强化学习版 Q-Learning）
- 基于策略：知道怎么行动
    - Policy Gradient（策略梯度方法）
         - REINFORCE
         - TRPO
         - PPO
         - DPG
         - Deterministic Policy Gradient
- 基于Actor-Critic（演员-评论家算法）：学生听老师，边行动边评价
    - A2C
    - A3C
    - PPO
    - DDPG
    - TD3
    - SAC
##### 1.3.5.2 按“学习方式”  
- 蒙特卡洛学习（MC Learning）：回合结束，用**真实回报**更新
- 时间差分学习（TD Learning）：不用等结束，用**估计值**边交互边更新
     - SARSA
     - Q-Learning
     - DQN
- 动态规划（Dynamic Programming）
     - Value Iteration（值迭代）
     - Policy Iteration（策略迭代）
##### 1.3.5.3 按“是否使用神经网络”
- 传统强化学习（RL）：不使用深度神经网络
     - Q-Learning
     - SARSA
     - Monte Carlo
- 深度强化学习（DRL）：深度学习 + 强化学习，用神经网络近似 Q函数 / V函数 / Policy函数
     - DQN
     - PPO
     - DDPG
     - SAC
##### 1.3.5.4 按“采样策略”
- 同策略（On-Policy）：用当前策略采样 + 学习
     - SARSA
     - REINFORCE
     - PPO
     - A2C
- 异策略（Off-Policy）：用其他策略采样，学习目标策略
     - Q-Learning
     - DQN
     - DDPG
     - TD3
     - SAC
#### 1.3.6 完整学习顺序
- 白盒环境强化学习
    - 马尔可夫决策过程
    - 动态规划
- 黑盒环境强化学习
    - 值函数估计
    - 无模型控制方法
- 参数化近似方法
    - 参数化值函数
    - 策略梯度
- 深度强化学习
    - 深度强化学习 - 价值方法
    - 深度强化学习 - 策略方法
### 1.4 深度强化学习
#### 1.4.1 方法

#### 1.4.2 目的

## 2 多臂老虎机（Multi-Armed Bandit）
无状态信息的强化学习，只有动作和奖励，用于研究“探索与利用”问题。
### 2.1 探索与利用（Exploration vs Exploitation）
- **利用（Exploitation）**：选择已知收益最高的动作
- **探索（Exploration）**：尝试未知动作，寻找更优策略

常用探索策略：
- $\epsilon$-greedy
- 衰减 $\epsilon$-greedy
- 乐观初始化
- 基于不确定性度量

### 2.2 Regret 理论
- 最优期望奖励：
$$Q^* = \max_{a} Q(a)$$
- Regret 定义： $R(a)=Q^*-Q(a)$
- 总 Regret：
$$\sigma_R = \mathbb{E}\left[\sum_{t=1}^T R(a_t)\right]$$
- 最优收敛速率： $O(\log T)$

### 2.3 增量式价值更新
$$Q_{n+1} = Q_n + \frac{1}{N(a)}(r_n - Q_n)$$
### 2.4 典型策略
- 贪心策略： $a^* = \arg\max Q(a)$
- $\epsilon$-greedy：以 $\epsilon$ 概率随机探索
- 衰减 $\epsilon$：可达到次线性 Regret


## 3 马尔可夫决策过程（MDP）
### 3.1 马尔可夫性质
未来只依赖当前状态，与历史无关：

$$P(S_{t+1}|S_t) = P(S_{t+1}|S_1,\dots,S_t)$$

### 3.2 MDP 五元组
$$(S,A,P_{sa},\gamma,R)$$
- $S$：状态空间
- $A$：动作空间
- $P_{sa}$：状态转移概率
- $\gamma \in [0,1]$：对未来奖励的折扣因子
- $R$：奖励函数

### 3.3 MDP 动态过程

$$s_0 \xrightarrow{a_0,R(s_0,a_0)} s_1 \xrightarrow{a_1,R(s_1,a_1)} s_2 \to \cdots$$

累计回报：

$$R_0 + \gamma R_1 + \gamma^2 R_2 + \cdots$$


## 4 基于模型的动态规划
### 4.1 状态价值函数 V

$$V^\pi(s) = \mathbb{E}_\pi\left[\sum_{k=0}^\infty \gamma^k R_{t+k+1} \big| S_t=s \right]$$

### 4.2 动作价值函数 Q

$$Q_\pi(s,a) = \mathbb{E}_\pi\left[ R + \gamma Q_\pi(s',a') \big| s,a \right]$$

### 4.3 Bellman 方程

$$V^\pi(s) = R(s) + \gamma \sum_{s'} P_{s\pi(s)}(s') V^\pi(s')$$

### 4.4 最优价值函数
- 最优价值函数是所有策略可获得的最大可能折扣奖励的和：

$$V^*(s) = \max_\pi V^\pi(s)$$

- 最优 Bellman 方程：

$$V^*(s) = R(s) + \max_{a \in A} \gamma \sum_{s'} P_{sa}(s') V^*(s')$$

- 最优策略：

$$V^*(s) = \arg\max_{a} \sum_{s'} P_{sa}(s') V^*(s')$$

### 4.5 动态规划算法：值迭代与策略迭代
#### 4.5.1 悬崖漫步环境
```python
import copy

class CliffWalkingEnv:
    """ 悬崖漫步环境 """
    def __init__(self, ncol=12, nrow=4):
        self.ncol = ncol  # 定义网格世界的列
        self.nrow = nrow  # 定义网格世界的行
        # 转移矩阵P[state][action] = [(p, next_state, reward, done)] 包含下一个状态和奖励
        self.P = self.createP()

    def createP(self):
        # 初始化
        P = [[[] for j in range(4)] for i in range(self.nrow * self.ncol)]
        # 4种动作，change[0]:上, change[1]:下, change[2]:左, change[3]:右。坐标系原点(0,0)定义在左上角
        change = [[0, -1], [0, 1], [-1, 0], [1, 0]]
        for i in range(self.nrow):
            for j in range(self.ncol):
                for a in range(4):
                    # 位置在悬崖或者目标状态，因为无法继续交互，任何动作奖励都为0
                    if i == self.nrow - 1 and j > 0:
                        P[i * self.ncol + j][a] = [(1, i * self.ncol + j, 0, True)]
                        continue
                    # 其他位置
                    next_x = min(self.ncol - 1, max(0, j + change[a][0]))
                    next_y = min(self.nrow - 1, max(0, i + change[a][1]))
                    next_state = next_y * self.ncol + next_x
                    reward = -1
                    done = False
                    # 下一个位置在悬崖或者终点
                    if next_y == self.nrow - 1 and next_x > 0:
                        done = True
                        if next_x != self.ncol - 1:  # 下一个位置在悬崖
                            reward = -100
                    P[i * self.ncol + j][a] = [(1, next_state, reward, done)]
        return P
```
#### 4.5.2 值迭代（Value Iteration）
直接迭代最优价值：
$$V(s) \leftarrow R(s) + \max_{a} \gamma \sum_{s'} P_{sa}(s') V(s')$$
特点：无显式策略，适合大状态空间。
实现代码：
```Python
class ValueIteration:
    """ 价值迭代算法 """
    def __init__(self, env, theta, gamma):
        self.env = env
        self.v = [0] * self.env.ncol * self.env.nrow  # 初始化价值为0
        self.theta = theta  # 价值收敛阈值
        self.gamma = gamma
        # 价值迭代结束后得到的策略
        self.pi = [None for i in range(self.env.ncol * self.env.nrow)]

    def value_iteration(self):
        cnt = 0
        while 1:
            max_diff = 0
            new_v = [0] * self.env.ncol * self.env.nrow
            for s in range(self.env.ncol * self.env.nrow):
                qsa_list = []  # 开始计算状态s下的所有Q(s,a)价值
                for a in range(4):
                    qsa = 0
                    for res in self.env.P[s][a]:
                        p, next_state, r, done = res
                        qsa += p * (r + self.gamma * self.v[next_state] * (1 - done))
                    qsa_list.append(qsa)  # 这一行和下一行代码是价值迭代和策略迭代的主要区别
                new_v[s] = max(qsa_list)
                max_diff = max(max_diff, abs(new_v[s] - self.v[s]))
            self.v = new_v
            if max_diff < self.theta: break  # 满足收敛条件,退出评估迭代
            cnt += 1
        print("价值迭代一共进行%d轮" % cnt)
        self.get_policy()

    def get_policy(self):  # 根据价值函数导出一个贪婪策略
        for s in range(self.env.nrow * self.env.ncol):
            qsa_list = []
            for a in range(4):
                qsa = 0
                for res in self.env.P[s][a]:
                    p, next_state, r, done = res
                    qsa += p * (r + self.gamma * self.v[next_state] * (1 - done))
                qsa_list.append(qsa)
            maxq = max(qsa_list)
            cntq = qsa_list.count(maxq)  # 计算有几个动作得到了最大的Q值
            # 让这些动作均分概率
            self.pi[s] = [1 / cntq if q == maxq else 0 for q in qsa_list]

# 主程序
env = CliffWalkingEnv()
action_meaning = ['^', 'v', '<', '>']
theta = 0.001
gamma = 0.9
agent = ValueIteration(env, theta, gamma)
agent.value_iteration()
print_agent(agent, action_meaning, list(range(37, 47)), [47])
```

#### 4.5.3 策略迭代（Policy Iteration）
两步循环：
1. **策略评估**：计算 $V^\pi$
2. **策略改进**： $\pi(s) = \arg\max_a \sum P_{sa}(s')V(s')$

特点：收敛快，适合小状态空间。
实现代码：
```python
class PolicyIteration:
    """ 策略迭代算法 """
    def __init__(self, env, theta, gamma):
        self.env = env
        self.v = [0] * self.env.ncol * self.env.nrow  # 初始化价值为0
        self.pi = [[0.25, 0.25, 0.25, 0.25] for i in range(self.env.ncol * self.env.nrow)]  # 初始化为均匀随机策略
        self.theta = theta  # 策略评估收敛阈值
        self.gamma = gamma  # 折扣因子

    def policy_evaluation(self):  # 策略评估
        cnt = 1  # 计数器
        while 1:
            max_diff = 0
            new_v = [0] * self.env.ncol * self.env.nrow
            for s in range(self.env.ncol * self.env.nrow):
                qsa_list = []  # 开始计算状态s下的所有Q(s,a)价值
                for a in range(4):
                    qsa = 0
                    for res in self.env.P[s][a]:
                        p, next_state, r, done = res
                        qsa += p * (r + self.gamma * self.v[next_state] * (1 - done))
                    qsa_list.append(self.pi[s][a] * qsa)
                new_v[s] = sum(qsa_list)  # 状态价值函数和动作价值函数之间的关系
                max_diff = max(max_diff, abs(new_v[s] - self.v[s]))
            self.v = new_v
            if max_diff < self.theta: break  # 满足收敛条件,退出评估迭代
            cnt += 1
        print("策略评估进行%d轮后完成" % cnt)

    def policy_improvement(self):  # 策略提升
        for s in range(self.env.nrow * self.env.ncol):
            qsa_list = []
            for a in range(4):
                qsa = 0
                for res in self.env.P[s][a]:
                    p, next_state, r, done = res
                    qsa += p * (r + self.gamma * self.v[next_state] * (1 - done))
                qsa_list.append(qsa)
            maxq = max(qsa_list)
            cntq = qsa_list.count(maxq)  # 计算有几个动作得到了最大的Q值
            # 让这些动作均分概率
            self.pi[s] = [1 / cntq if q == maxq else 0 for q in qsa_list]
        print("策略提升完成")
        return self.pi

    def policy_iteration(self):  # 策略迭代
        while 1:
            self.policy_evaluation()
            old_pi = copy.deepcopy(self.pi)  # 将列表进行深拷贝,方便接下来进行比较
            new_pi = self.policy_improvement()
            if old_pi == new_pi: break


def print_agent(agent, action_meaning, disaster=[], end=[]):
    print("状态价值: ")
    for i in range(agent.env.nrow):
        for j in range(agent.env.ncol):
            # 为了输出美观,保持输出6个字符
            print('%6.6s' % ('%.3f' % agent.v[i * agent.env.ncol + j]), end=' ')
        print()

    print("策略: ")
    for i in range(agent.env.nrow):
        for j in range(agent.env.ncol):
            # 一些特殊的状态,例如悬崖漫步中的悬崖
            if (i * agent.env.ncol + j) in disaster:
                print('*****', end=' ')
            elif (i * agent.env.ncol + j) in end:  # 目标状态
                print('EEEE', end=' ')
            else:
                a = agent.pi[i * agent.env.ncol + j]
                pi_str = ''
                for k in range(len(action_meaning)):
                    pi_str += action_meaning[k] if a[k] > 0 else 'o'
                print(pi_str, end=' ')
        print()


# 主程序
env = CliffWalkingEnv()
action_meaning = ['^', 'v', '<', '>']
theta = 0.001
gamma = 0.9
agent = PolicyIteration(env, theta, gamma)
agent.policy_iteration()
print_agent(agent, action_meaning, list(range(37, 47)), [47])
```
#### 4.5.4 对比总结
- 价值迭代：直接更新价值，一步求最优
- 策略迭代：先评估再改进，更稳定
- 均基于 Bellman 最优方程，可收敛到全局最优

## 5 MC学习、TD学习、SARSA与Q学习
### 5.1 无模型强化学习
- **背景**：现实问题中，环境的状态转移概率 $P_{s's}$ 和奖励函数 $R(s,a)$ 通常无法直接获得。
- **数据形式**：仅能通过与环境交互，观察到完整的「片段（Episodes）」
  - Episode 1: $s_0^{(1)} \xrightarrow[a_0^{(1)}]{R(s_0)^{(1)}} s_1^{(1)} \xrightarrow[a_1^{(1)}]{R(s_1)^{(1)}} s_2^{(1)} \dots s_T^{(1)}$
  - Episode 2: $s_0^{(2)} \xrightarrow[a_0^{(2)}]{R(s_0)^{(2)}} s_1^{(2)} \xrightarrow[a_1^{(2)}]{R(s_1)^{(2)}} s_2^{(2)} \dots s_T^{(2)}$
- **核心思想**：不依赖构建MDP模型，直接从交互经验中学习值函数（Value）和策略（Policy）。
- **关键步骤**：
  1. 估计值函数（Policy Evaluation）
  2. 优化策略（Policy Improvement）
  
### 5.2 值函数估计的两种路径
#### 5.2.1 基于模型的强化学习（MDP已知）
值函数可通过动态规划直接求解：

$$
\begin{aligned}
V^\pi(s) &= \mathbb{E}\left[ R(s_0) + \gamma R(s_1) + \gamma^2 R(s_2) + \dots \mid s_0 = s, \pi \right] \\
&= R(s) + \gamma \sum_{s' \in S} P_{s\pi(s)}(s') V^\pi(s')
\end{aligned}
$$

#### 5.2.2 无模型强化学习（MDP未知）
- 无法直接获取 $P_{sa}$ 和 $R(s,a)$
- 替代方案：利用交互得到的经验片段，通过蒙特卡洛（MC）或时间差分（TD）方法估计值函数

### 5.3 蒙特卡洛（Monte-Carlo, MC）方法原理
#### 5.3.1 定义
- 一类依赖**重复随机抽样**来获得数值结果的计算方法。
#### 5.3.2 蒙特卡洛价值估计
目标：从策略 $\pi$ 生成的经验片段中，估计策略值函数 $V^\pi(s)$。
累计奖励（Return）：

$$G_t = R_{t+1} + \gamma R_{t+2} + \dots + \gamma^{T-t-1} R_T$$

值函数的定义是**期望累计奖励**：

$$V^\pi(s) = \mathbb{E}[G_t \mid s_t = s, \pi]$$

蒙特卡洛方法用**经验均值**替代期望：

$$
V^\pi(s) \approx \frac{1}{N} \sum_{i=1}^N G_t^{(i)}
$$

- $G_t^{(i)}$ ：第 $i$ 个片段中，状态 $s$ 在时间步 $t$ 对应的累计奖励
- 当 $N \to \infty$ 时，由大数定律， $V(s) \to V^\pi(s)$
#### 5.3.3 蒙特卡洛价值估计的实现
##### 5.3.3.1 批量实现（非增量）
1.  使用策略 $\pi$ 采样多个完整片段
2.  对每个片段中的状态 $s$：
    - 计数器：
      $N(s) \leftarrow N(s) + 1$
      
    - 累计奖励和：
      $S(s) \leftarrow S(s) + G_t$
      
    - 值估计：
      $V(s) = \frac{S(s)}{N(s)}$
      
##### 5.3.3.2 增量实现（Incremental MC）
每个片段结束后，逐步更新值函数：
1.  对每个状态 $S_t$ 及其累计奖励 $G_t$：
   
    $N(S_t) \leftarrow N(S_t) + 1$
    
    $V(S_t) \leftarrow V(S_t) + \frac{1}{N(S_t)} \left( G_t - V(S_t) \right)$
    
2.  对于非稳定环境（环境随时间变化），使用固定学习率跟踪近期经验：
   
    $V(S_t) \leftarrow V(S_t) + \alpha \left( G_t - V(S_t) \right)$
    
### 5.4 时序差分（TD）学习基础
#### 5.4.1 核心定义与更新公式
TD学习是无模型强化学习的核心方法之一，核心思想是**用“自举（bootstrapping）”方式，用下一时刻的估计值更新当前状态价值**，无需等待回合结束。

- 累计回报（定义）：
  
  $$G_t = R_{t+1} + \gamma R_{t+2} + \gamma^2 R_{t+3} + \dots$$
  
- TD 近似回报（估计值）：

  $$G_t \approx R_{t+1} + \gamma V(S_{t+1})$$
  
- TD 状态价值更新公式：

  $$V(S_t) \leftarrow V(S_t) + \alpha \left( R_{t+1} + \gamma V(S_{t+1}) - V(S_t) \right)$$
  
  - 其中： $R_{t+1}$ 是观测到的即时奖励， $V(S_{t+1})$ 是对未来的估计（自举）。
#### 5.4.2 特点
- 直接从经验片段中学习，属于**无模型**强化学习
- 无需预先知道MDP的状态转移和奖励函数
- 支持**从不完整的片段中学习**（无需等待回合结束）
- 可以**在线学习**，每一步交互后都能更新
- 适用于连续、无终止的环境（MC无法处理）

### 5.5 MC学习 vs TD 学习
#### 5.5.1 核心目标
两者目标一致：从策略 $\pi$ 生成的经验片段中学习状态价值函数 $V^\pi(s)$ 。

#### 5.5.2 更新方式对比

| 方法 | 更新目标 | 公式 | 特点 |
| :--- | :--- | :--- | :--- |
| 蒙特卡洛（MC） | 真实累计回报 $G_t$ | $$V(S_t) \leftarrow V(S_t) + \alpha \left( G_t - V(S_t) \right)$$ | 无偏估计，方差高，必须回合结束才能更新 |
| 时序差分（TD） | 估计累计回报 $R_{t+1} + \gamma V(S_{t+1})$ | $$V(S_t) \leftarrow V(S_t) + \alpha \left( R_{t+1} + \gamma V(S_{t+1}) - V(S_t) \right)$$ | 有偏估计，方差低，可单步更新 |

TD学习的更新误差为：

$$\delta_t = R_{t+1} + \gamma V(S_{t+1}) - V(S_t)$$

#### 5.5.3 优缺点对比
| 维度    | 蒙特卡洛（MC）         | 时序差分（TD）          |
| :---- | :--------------- | :---------------- |
| 更新时机  | 必须等待片段结束         | 每一步之后即可在线更新       |
| 数据要求  | 只能从完整序列学习        | 可从不完整序列学习         |
| 适用环境  | 仅片段化、有终止的环境      | 可在连续无终止环境中工作      |
| 偏差与方差 | 无偏，高方差           | 有偏，低方差            |
| 收敛性   | 对初始值不敏感，函数近似下仍收敛 | 通常更高效，但函数近似下不一定收敛 |
| 理解难度  | 简单直观             | 依赖自举，理解稍复杂        |
### 5.6 SARSA 算法（同策略 TD 控制）
#### 5.6.1 核心思想
SARSA 是基于TD学习的**同策略（on-policy）**控制算法，直接学习状态-动作价值函数 $Q(s,a)$，更新依赖当前策略生成的五元组：

$$(S, A, R, S', A')$$

#### 5.6.2 更新公式

$$Q(S, A) \leftarrow Q(S, A) + \alpha \left( R + \gamma Q(S', A') - Q(S, A) \right)$$

- 其中：$A$ 和 $A'$ 均由当前策略（如 $\epsilon$-贪心）选择。

#### 5.6.3 算法流程（伪代码）

- 初始化：对所有状态 $s ∈ S$、动作 $a ∈ A(s)$，任意初始化状态-动作值函数 $Q(s,a)$；终止状态的所有动作值设为 $0$。

- 循环（对每一幕/回合）：

    - 初始化当前状态 $S$
    
    - 基于当前的 $Q$ 值，用策略（如 $ε-greedy$）从状态 S 中选择动作 A
    
    - 循环（对回合中的每一步）：
    
        - 执行动作 $A$，观测到即时奖励 R 和下一个状态 S'
        
        - 基于当前的 $Q$ 值，用同样的策略（如 $ε-greedy$）从状态 S' 中选择下一个动作 $A'$
        
        - 更新状态-动作值函数：
        
            $Q(S, A) ← Q(S, A) + α × [ R + γ × Q(S', A') − Q(S, A) ]$
            
        - 更新状态和动作：
        
            $S ← S'$
            
            $A ← A'$
            
    - 直到状态 $S$ 到达终止状态
    
#### 5.6.4 关键特点
- **同策略（On-Policy）**：两个动作 $A$ 和 $A'$ 均由当前策略生成
- 基于TD学习，单步更新，无需等待回合结束
- 适用于在线交互环境，数据利用率高

### 5.7 Q学习（Q-Learning）
#### 5.7.1 核心思想
学习每个状态—动作对的价值（$Q$ 值），并选择价值最大的动作。
#### 5.7.2 特点

|**特点**|**内容**|
|---|---|
|强化学习类型|Value-Based|
|学习方式|TD Learning|
|策略类型|Off-Policy|
|是否需要环境模型|不需要|
|是否适合离散动作|非常适合|
|是否适合连续动作|不适合|

#### 5.7.3 Q函数（动作价值函数）

$$Q(s,a)=\mathbb{E}[r_t+\gamma r_{t+1}+\gamma^2 r_{t+2}+\cdots]$$

含义：从状态 $s$ 执行动作 $a$ 后，未来累计奖励的期望

#### 5.7.4 最优Q函数

$$Q^\*(s,a)=\max_\pi Q^\pi(s,a)$$

即：所有策略中最大的 $Q$ 值
#### 5.7.5 Bellman 最优方程

$$Q^*(s,a)=r+\gamma\max_{a'}Q^*(s',a')$$

当前 $Q$ 值：即时奖励+未来最大收益
#### 5.7.6 Q-Learning 更新公式

$$Q(s,a)\leftarrow Q(s,a)+\alpha[r+\gamma\max_{a'}Q(s',a')-Q(s,a)]$$

| **部分**         | **含义**   |
| -------------- | -------- |
| $Q(s,a)$       | 当前估计     |
| $r$            | 当前奖励     |
| $γ$            | 折扣因子     |
| $max Q(s’,a’)$ | 下一状态最大收益 |
| $α$            | 学习率      |

#### 5.7.7 TD误差（Temporal Difference Error）

$$\delta=r+\gamma\max_{a'}Q(s',a')-Q(s,a)$$

#### 5.7.8 Q-Learning 算法流程
初始化 $Q$ 表

重复训练：
        1.获取当前状态 s
        2.根据策略选择动作 a     
        3.执行动作      
        4.获得奖励 r 和新状态 s'       
        5.更新 Q 值      
        6.状态转移 s ← s'
        
#### 5.7.9 ε-greedy 策略
##### 5.7.9.1 规则
以 $ε$ 概率随机探索,以 $1-ε$ 概率选择最大 $Q$ 值
##### 5.7.9.2 表达式

$$a=\begin{cases}\text{random action}, & \text{with probability }\epsilon\\\arg\max_a Q(s,a), & \text{with probability }1-\epsilon\end{cases}$$

#### 5.7.10 Q-Learning 优缺点
##### 5.7.10.1 优点

|**优点**|**内容**|
|---|---|
|简单|易实现|
|理论成熟|收敛性较好|
|无模型|不需要环境模型|

##### 5.7.10.2 缺点

| **缺点**  | **内容**        |
| ------- | ------------- |
| 维度灾难    | 状态空间过大时无法使用表格 |
| 不适合连续动作 | max操作难以求解     |
| 样本效率低   | 训练速度慢         |
