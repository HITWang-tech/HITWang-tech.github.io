## 1 阶段一：大模型工程化基础
大模型应用开发本质还是后端工程。掌握 `并发` 、`异步`、`容器化`是后续优化大模型接口响应速度的前提。
Python 异步并发 → FastAPI → Docker/Redis地基打好，后面学 RAG、Agent 你会完全通透,知道每一步底层在干嘛，而不是单纯复制别人代码跑 Demo。
### 1.1 Python 高级与并发编程
大模型的 API 调用通常是 IO 密集型任务，掌握异步极其重要。
- **核心知识点**：多线程(Threading)、多进程(Multiprocessing)、协程(`asyncio`)、生成器与流式输出(`yield`)。
- **视频推荐**：【2021最新版】Python 并发编程实战
  [https://www.bilibili.com/video/BV1bK411A7tV](https://www.bilibili.com/video/BV1bK411A7tV)
- **书籍推荐**：《流畅的 Python》(Fluent Python)
  [https://book.douban.com/subject/27028517/](https://book.douban.com/subject/27028517/)

### 1.2 后端框架与工程基建
- **Web 框架 FastAPI**：现代大模型应用首选。必须掌握路由、依赖注入、以及 SSE (Server-Sent Events) 用于实现类似 ChatGPT 的打字机流式输出效果。
  - 官方文档：
   [https://fastapi.tiangolo.com/zh/](https://fastapi.tiangolo.com/zh/)
- **工程化基建组件**
  - Docker：(环境隔离，一键部署模型和应用必学)
    [https://docs.docker.com/](https://docs.docker.com/)
  - Redis：用于对话记忆(Memory)的高速缓存、分布式锁。
  - Celery / RabbitMQ：用于处理后台极度耗时的任务（如百兆PDF的OCR解析与向量化）。

### 1.3 优秀开源实战项目 (RAG & Agent 练手)
在打好工程地基的同时，直接阅读和运行优秀的开源实战项目是提升最快的方式。以下是几个非常适合练手和二次开发的 GitHub 项目：

#### 1.3.1 RAG实战练手项目
- Langchain-Chatchat：国内最火的基于本地知识库的 LLM 问答项目。代码结构清晰，非常适合用来学习如何将 RAG 落地到实际企业级 Web 应用中。
  [https://github.com/chatchat-space/Langchain-Chatchat](https://github.com/chatchat-space/Langchain-Chatchat)
- privateGPT：极简的本地私有化 RAG 实现，可以学习如何完全离线处理文档和构建本地向量检索链路。
  [https://github.com/imartinez/privateGPT](https://github.com/imartinez/privateGPT)
- Quivr：你的生成式 AI “第二大脑”，基于 RAG 的云端个人知识库，带有完善的前后端界面，适合全栈练手。
  [https://github.com/QuivrHQ/quivr](https://github.com/QuivrHQ/quivr)

#### 1.3.2 Agent实战练手项目
- ChatDev：基于多智能体协同的虚拟软件公司。你可以看到 Agent 如何扮演 CEO、程序员、测试员并协同写代码。极佳的多 Agent 学习范例。
  [https://github.com/OpenBMB/ChatDev](https://github.com/OpenBMB/ChatDev)
- Open Interpreter：让大模型直接在你的终端上执行代码（Python, Shell 等）。学习 Tool Calling 和 代码解释器(Code Interpreter) 底层机制的必读项目。
  [https://github.com/OpenInterpreter/open-interpreter](https://github.com/OpenInterpreter/open-interpreter)
- LLM Universe (Datawhale)：专为初学者设计的面向开发者的 LLM 入门实战教程，包含多个可以直接跑通的 RAG 和 Agent 小 Demo，极度友好。
  [https://github.com/datawhalechina/llm-universe](https://github.com/datawhalechina/llm-universe)

## 2 阶段二：RAG (检索增强生成) 深度拆解与高阶应用
RAG 不是简单的 "切块 -> 存向量 -> 搜索 -> 丢给模型"。在工业界，数据的清洗解析、检索的自适应判断才是核心壁垒。

### 2.1 RAG 基础核心框架
- LangChain：最通用的 LLM 编排框架，包含极其丰富的 Document Loaders（文档加载器）。
  - 官方文档：[https://python.langchain.com/docs/introduction/](https://python.langchain.com/docs/introduction/)
  - 视频(快速入门)：[https://www.bilibili.com/video/BV1cCd6YwE4n](https://www.bilibili.com/video/BV1cCd6YwE4n)
- LlamaIndex：专为 RAG 诞生，在索引构建、高级检索策略上比 LangChain 更好用。
  - 官方文档：[https://docs.llamaindex.ai/](https://docs.llamaindex.ai/)

### 2.2 向量数据库与底层索引算法
- **底层算法（面试必问）**：HNSW (分层导航小世界算法 - 目前最主流的近似近邻搜索)、IVF-Flat。
- **向量库工具**
  - Milvus：阿里/字节等大厂常用的企业级分布式向量库，适合海量数据。
    [https://milvus.io/](https://milvus.io/)
  - Qdrant：Rust 编写，性能极高，支持极其复杂的 Payload 标量过滤。
    [https://qdrant.tech/](https://qdrant.tech/)
  - Chroma / FAISS：轻量级，本地开发和中小型项目首选。

### 2.3 高阶 RAG 核心优化策略 ⭐ 面试/实战绝对重点
1. **文档解析与结构化切块 (Document Parsing & Chunking)**
   - 痛点：PDF 中的表格、复杂排版直接切块会变成乱码。
   - 进阶方案：Markdown 感知切块（将文档先转为MD再按标题切）、语义切块(Semantic Chunking)、父子文档检索(Auto-merging)。
   - 利器：
     - LlamaParse：极其强大的复杂 PDF 解析 API。
       [https://github.com/run-llama/llama_parse](https://github.com/run-llama/llama_parse)
     - Unstructured：工业级非结构化数据ETL工具。
       [https://github.com/Unstructured-IO/unstructured](https://github.com/Unstructured-IO/unstructured)

2. **检索中优化 (Retrieval Strategies)**
   - 查询重写 (Query Rewrite) / 多路召回：用户的提问往往很短，先用 LLM 扩写成多个方向的 Query。
   - HyDE (假设性文档嵌入)：让大模型先盲答一段（包含幻觉没关系），然后拿这篇“假答案”的向量去搜真实的文档，召回率奇高。
   - 混合检索 (Hybrid Search)：向量检索 (懂语义) + BM25 关键词检索 (懂专有名词，如型号、人名)，两者结合加权。

3. **自反思与动态 RAG (Self-Correction RAG)**
   - Self-RAG / CRAG：检索出来的东西不能直接用！要先用一个小模型判断 “检索内容是否与问题相关？”。如果无关，触发网络搜索 (Web Search) 补充，或者重新检索。

4. **检索后优化 (Post-Retrieval)**
   - 重排 (Reranking)：初筛出 Top 20，用 Reranker 模型二次精准打分，只留 Top 5。
   - BGE-Reranker：[https://github.com/FlagOpen/FlagEmbedding](https://github.com/FlagOpen/FlagEmbedding)

### 2.4 RAG 前沿范式与工程框架
- DSPy (自动 Prompt 优化框架)：斯坦福出品。不要再手写 Prompt 了！用写代码的逻辑写 RAG 管道，DSPy 会自动帮你优化每一个节点的 Prompt。
  [https://github.com/stanfordnlp/dspy](https://github.com/stanfordnlp/dspy)
- RAGFlow：字节跳动开源。提供全流程可视化工具链，解决了很多框架搞不定的 “复杂排版/表格 PDF 的深度解析” 问题。
  [https://github.com/infiniflow/ragflow](https://github.com/infiniflow/ragflow)
- GraphRAG：微软开源。纯向量 RAG 无法回答 “请总结这本红楼梦里所有人物的宏观关系”，引入知识图谱 (Knowledge Graph) 解决全局推理问题。
  [https://github.com/microsoft/graphrag](https://github.com/microsoft/graphrag)
- Ragas / TruLens：评测 RAG 的核心指标（忠实度/防幻觉、回答相关性）。
  - Ragas 链接：[https://docs.ragas.io/](https://docs.ragas.io/)

## 3 阶段三：Agent (智能体) 开发与底层机制
Agent 是当前大模型最前沿的落地形态。不要迷信框架，要深刻理解 Agent 的四大底层基石：**记忆、规划、工具、角色**。

### 3.1 Agent 四大核心组件深度拆解
#### 3.1.1 Profile (角色与人设工程)
不仅是给个 System Prompt，还要包含它的工作流定义、边界限制（不能回答什么）、输出格式强制规范 (JSON Schema)。

#### 3.1.2 ⭐ Memory (记忆机制 - Agent 的灵魂)
Agent 如果像鱼一样只有7秒记忆，永远做不了复杂任务。记忆分为以下几个层级：
- **短期记忆 (Short-term Memory)**：当前的对话历史 (Message History)。需要掌握上下文窗口管理 (Token 截断、滑动窗口、对话总结)。
- **长期记忆 (Long-term Memory)**
  - 情景记忆 (Episodic Memory)：记录 Agent 过去做过的事和解决过的问题（例如：“我昨天尝试调用 API 失败了，是因为缺少 Token”），存入向量库供未来参考。
  - 语义记忆 (Semantic Memory)：提取出关于世界或用户的客观事实（例如：“用户的名字叫张三，喜欢 Python”）。
- **记忆框架推荐**
  - Mem0 (原 Embedchain)：目前最火的 Agent 记忆层实现，专为长上下文和个性化设计。
    [https://github.com/mem0ai/mem0](https://github.com/mem0ai/mem0)
  - Zep：专为大语言模型应用构建的快速、可扩展的长期记忆服务。
    [https://github.com/getzep/zep](https://github.com/getzep/zep)
  - LangMem：LangChain 官方出的记忆模块提取工具。

#### 3.1.3 Planning (规划与推理)
面对复杂任务，Agent 需要先想后做。
- ReAct 范式：思考 (Thought) -> 动作 (Action) -> 观察 (Observation) 的循环。
- ToT (Tree of Thoughts - 思维树)：面对复杂问题（如解数学题、写小说），探索多种可能的分支，自我评估后选择最优路径。
- Self-Refine (自我反思)：模型输出结果后，强迫它以 Critic (评论员) 视角检查自己的错误并重写。

#### 3.1.4 Action & Tools (工具调用与协议)
- Function Calling (函数调用)：OpenAI 提出的标准，大模型输出应该调用什么函数以及参数，由本地代码执行后将结果返回给大模型。
- ⭐ MCP 协议 (Model Context Protocol)：Anthropic (Claude) 最新开源的标准协议！用于统一不同的数据源和工具接口。以后 Agent 接入数据库、接入飞书、接入 Github，只需要支持 MCP 协议即可，不用再手写无数的 API 对接代码。
  - 视频介绍：[https://www.bilibili.com/video/BV1qTYizcEN3](https://www.bilibili.com/video/BV1qTYizcEN3)

### 3.2 多智能体协作模式 (Multi-Agent Patterns) 与主流框架
单兵作战能力有限，未来的系统是多 Agent 协同的。
- **核心协作模式 (架构设计)**
  - Supervisor (主管模式)：一个 LLM 作为主管，负责将任务拆解并分发给下面不同的专业 Agent（如画图Agent、写代码Agent），最后由主管汇总。
  - Hierarchical (层级模式)：树状结构，大包工头分给小包工头。
  - Swarm (蜂群模式)：Agent 之间点对点平级交流，互相交接任务 (Handoffs)。
- **主流实战框架**
  - LangGraph：基于图 (Graph) 和状态机编排，对数据流的控制极度严谨，最适合企业级落地。
    [https://langchain-ai.github.io/langgraph/](https://langchain-ai.github.io/langgraph/)
  - CrewAI：基于 “角色扮演” 的多智能体框架。像开公司一样定义角色和任务，上手极快。
    [https://www.crewai.com/](https://www.crewai.com/)
  - OpenAI Swarm：OpenAI 官方开源的超轻量级多智能体实验框架，主打 Routines 和 Handoffs (交接) 机制。
    [https://github.com/openai/swarm](https://github.com/openai/swarm)
  - MetaGPT：国内顶尖框架，模拟软件公司的标准作业流程 (SOP)。
    [https://github.com/geekan/MetaGPT](https://github.com/geekan/MetaGPT)

### 3.3 高阶深水区：Agent-RL (智能体强化学习) 与对齐
突破 Prompt 极限，用强化学习训练天生具备 Agent 能力的模型。
- **核心理论**：RLHF(基于人类反馈) -> DPO (直接偏好优化，抛弃复杂奖励模型)。
- **Agent 交互式训练**：让 Agent 在真实的模拟环境中去试错。
  - WebArena / OSWorld：模拟真实网页/操作系统的评测与训练环境。
    [https://webarena.dev/](https://webarena.dev/)
  - TRL (Transformer Reinforcement Learning)：Hugging Face 官方强化学习库，支持 PPO，DPO 等全套流程，想自己训练对齐模型的必学库。
    [https://huggingface.co/docs/trl/index](https://huggingface.co/docs/trl/index)

## 4 阶段四：加分项--模型微调 (Fine-Tuning) 与 高性能部署
### 4.1 高效微调技术 (PEFT)
- **核心算法**：LoRA (低秩适应)，QLoRA (量化微调)等。
- **常见微调框架**
  - LLaMA-Factory：提供零代码可视化的 WebUI 界面，支持各种微调和 DPO 评测。
    [https://github.com/hiyouga/LLaMA-Factory](https://github.com/hiyouga/LLaMA-Factory)
  - Unsloth：显存降低 70%，速度提升两倍的微调框架。
    [https://github.com/unslothai/unsloth](https://github.com/unslothai/unsloth)

### 4.2 推理加速与本地部署 (Inference)
- 本地小白极速体验：Ollama
  [https://ollama.com/](https://ollama.com/)
- **企业级高并发推理引擎**
  - vLLM：当前工业界绝对主流。必须了解其底层的 PagedAttention (分页注意力) 和 KV Cache 机制。
    [https://github.com/vllm-project/vllm](https://github.com/vllm-project/vllm])
  - SGLang：针对结构化输出和复杂 Prompt 友好的推理引擎。
